# 导表工具使用说明

本文档面向项目协作者，说明当前 Excel 导表工具的完整使用流程，包括：

1. Excel 放置位置与表结构约定
2. Unity 编辑器内如何执行导出
3. 导出后会生成哪些资源和代码
4. 运行时如何通过 `LoadData` 加载表数据
5. 常见限制、报错原因与协作注意事项

## 一、工具概览

当前项目内置了一套基于 Unity Editor 的 Excel 导表工具，入口位于：

- 菜单：`Tools/Export Tables`

它会将 `Assets/Resources/Docs` 目录下的 `.xlsx` 文件解析为 JSON，并生成一份总的数据类型定义文件，供运行时直接加载。

固定目录如下：

- Excel 根目录：`Assets/Resources/Docs`
- JSON 导出目录：`Assets/Resources/GameData/ExportedTables`
- 生成代码目录：`Assets/Scripts/GameData/GameDataTables.cs`

这三个路径是工具当前的固定约定，不是自由配置项。

## 二、推荐协作流程

推荐所有协作者都遵循下面的顺序：

1. 在 `Assets/Resources/Docs` 下新增或修改 `.xlsx` 文件
2. 保证每个 Sheet 的表头格式符合工具约定
3. 在 Unity 中打开 `Tools/Export Tables`
4. 选择要导出的文件夹或具体 `.xlsx` 文件
5. 点击“导出”
6. 检查 `Assets/Resources/GameData/ExportedTables` 和 `Assets/Scripts/GameData/GameDataTables.cs` 是否更新
7. 在代码中通过 `LoadData.LoadJSONData<T>()` 使用导出的数据

如果 Excel 改了但没有重新导出，运行时代码不会自动读取到 Excel 的新内容，只会读取上次导出的 JSON。

## 三、Excel 文件放置规则

只支持放在：

- `Assets/Resources/Docs`

及其子目录中的 `.xlsx` 文件。

不支持：

- `.xls`
- `.csv`
- 根目录之外的 Excel 文件

工具窗口中也只会显示 `Assets/Resources/Docs` 下的文件夹和 `.xlsx` 文件。

## 四、单个 Sheet 的结构约定

每个可导出的 Sheet 至少需要 4 行：

1. 第 1 行：字段说明
2. 第 2 行：字段类型
3. 第 3 行：字段名
4. 第 4 行起：数据行

如果一个 Sheet 少于 4 行，会被直接跳过，不会导出。

### 4.1 行含义说明

#### 第 1 行：字段说明

这行主要用于给人看，也会写入导出的 JSON 的 `columns.description`。

示例：

| ID | 名称 | 价格 | 是否启用 |
| --- | --- | --- | --- |

#### 第 2 行：字段类型

这行决定生成的 C# 字段类型和 JSON 序列化规则。

当前支持的类型只有：

- `int`
- `int32`
- `long`
- `int64`
- `float`
- `single`
- `double`
- `bool`
- `boolean`
- `string`

其中工具会自动归一化：

- `int32` -> `int`
- `int64` -> `long`
- `single` -> `float`
- `boolean` -> `bool`

如果写了不支持的类型，例如 `List<int>`、`enum`、`decimal`、`vector3`，导出会直接报错。

#### 第 3 行：字段名

这行用于生成 C# 字段名，并作为 JSON 的字段键名。

建议：

- 使用英文
- 使用明确的业务名
- 避免空格和特殊字符
- 一旦对外使用，尽量不要随意改名

#### 第 4 行起：数据

从第 4 行开始，每一行都会被视为一条记录。

空行会被跳过。

如果整个 Sheet 没有任何有效数据行，也会被跳过，不会导出。

### 4.2 一个完整示例

假设 Excel 文件 `Items.xlsx` 中有一个 Sheet 叫 `Item`，内容如下：

| ID | 名称 | 价格 | 是否启用 |
| --- | --- | --- | --- |
| int | string | float | bool |
| id | name | price | enabled |
| 1 | Sword | 10.5 | true |
| 2 | Shield | 20 | false |

导出后会生成：

- 行结构：`ItemRecord`
- 表结构：`ItemTable`

并输出一份类似下面结构的 JSON：

```json
{
  "tableName": "Item",
  "sourceFile": "Items.xlsx",
  "sourceRelativePath": "Items.xlsx",
  "sheetName": "Item",
  "rowCount": 2,
  "columns": [
    { "name": "id", "type": "int", "description": "ID" },
    { "name": "name", "type": "string", "description": "名称" },
    { "name": "price", "type": "float", "description": "价格" },
    { "name": "enabled", "type": "bool", "description": "是否启用" }
  ],
  "data": [
    { "id": 1, "name": "Sword", "price": 10.5, "enabled": true },
    { "id": 2, "name": "Shield", "price": 20.0, "enabled": false }
  ]
}
```

## 五、Sheet 名与生成类型的关系

工具以 Sheet 名作为逻辑表名。

例如：

- Excel 文件：`Items.xlsx`
- Sheet 名：`Item`

则生成：

- `ItemRecord`
- `ItemTable`

如果另一个 Sheet 名叫 `Text`，则会生成：

- `TextRecord`
- `TextTable`

这里要特别注意：

- 文件名主要影响导出的 JSON 文件名
- Sheet 名决定逻辑表名和生成的 C# 类型名

也就是说，同一个 Excel 中的不同 Sheet，会被视为不同表。

## 六、编辑器内如何导出

### 6.1 打开工具

在 Unity 顶部菜单中打开：

- `Tools/Export Tables`

会弹出“导出表格”窗口。

### 6.2 选择导出范围

工具窗口左侧树会展示 `Assets/Resources/Docs` 下的内容。你可以选择：

- 整个根目录
- 某个子文件夹
- 某个具体的 `.xlsx` 文件

导出范围含义如下：

- 选择根目录：导出全部 Excel
- 选择文件夹：只导出该文件夹及其子目录下的 Excel
- 选择单个 `.xlsx`：只导出该文件

这对于协作时非常重要。只改了一张表时，可以只导出对应文件，不需要每次全量导出。

### 6.3 点击导出

点击窗口底部的“导出”按钮后，工具会执行：

1. 扫描当前选中的范围
2. 解析所有可导出的 Sheet
3. 生成或更新对应 JSON
4. 生成或更新 `tables_manifest.json`
5. 重新生成 `Assets/Scripts/GameData/GameDataTables.cs`
6. 调用 Unity 的 `AssetDatabase.Refresh()`

如果成功，窗口里会显示类似信息：

- 写入了多少文件
- 删除了多少旧文件
- 跳过了多少 Sheet

同时 Console 中也会打印解析日志。

## 七、导出产物说明

### 7.1 JSON 文件

输出目录固定为：

- `Assets/Resources/GameData/ExportedTables`

JSON 文件名规则为：

- `Excel文件名_Sheet名.json`

例如：

- `Items.xlsx` 的 `Item` Sheet
- 导出为 `Items_Item.json`

如果 Excel 位于 `Docs` 下的子目录中，导出目录会保持相同的相对目录结构。

例如：

- Excel：`Assets/Resources/Docs/Chapter1/Items.xlsx`
- JSON：`Assets/Resources/GameData/ExportedTables/Chapter1/Items_Item.json`

### 7.2 Manifest 文件

导出目录中还会生成：

- `tables_manifest.json`

它记录了所有已导出表的索引信息，包括：

- `tableName`
- `sourceFile`
- `sourceRelativePath`
- `sheetName`
- `outputRelativePath`
- `rowCount`

运行时 `LoadData` 先读取它，再根据 `tableName` 找到对应 JSON。

### 7.3 自动生成代码

工具每次导出后，都会重写：

- `Assets/Scripts/GameData/GameDataTables.cs`

这个文件会为当前所有导出的表生成 C# 类型定义。

例如：

```csharp
[Serializable]
public class ItemRecord
{
    public int id;
    public string name;
    public float price;
    public bool enabled;
}

[Serializable]
public class ItemTable
{
    public string tableName;
    public string sourceFile;
    public string sourceRelativePath;
    public string sheetName;
    public int rowCount;
    public List<ItemRecord> data;
}
```

注意：

- `GameDataTables.cs` 属于生成文件
- 不建议手动修改
- 手动修改会在下次导出时被覆盖

## 八、运行时如何加载数据

运行时统一使用：

```csharp
LoadData.LoadJSONData<T>(string tableName)
```

其中：

- `T` 通常是导表生成的 `XXXTable`
- `tableName` 是逻辑表名，也就是 Sheet 名

### 8.1 基本示例

```csharp
ItemTable itemTable = LoadData.LoadJSONData<ItemTable>("Item");
if (itemTable == null)
{
    Debug.LogError("Item 表加载失败");
    return;
}

for (int i = 0; i < itemTable.data.Count; i++)
{
    ItemRecord row = itemTable.data[i];
    Debug.Log($"id={row.id}, name={row.name}, price={row.price}, enabled={row.enabled}");
}
```

### 8.2 关键点

- 泛型类型要和导出的表结构匹配，例如 `ItemTable`
- 传入的 `tableName` 要和 Sheet 名一致，例如 `Item`
- `tableName` 大小写在内部按忽略大小写处理，但协作时仍建议统一写正确名称

如果你传错类型，例如：

```csharp
TextTable table = LoadData.LoadJSONData<TextTable>("Item");
```

或者自己定义了一个和导出字段不匹配的类型，`LoadData` 会在校验时直接报错并返回 `null`。

## 九、`LoadData` 的校验与缓存行为

`LoadData` 不只是简单反序列化，它还会做结构校验。

### 9.1 它会校验什么

主要包括：

- 目标类型中必须存在 `tableName` 字段
- 目标类型中必须存在 `data` 字段
- `tableName` 必须和请求加载的表名一致
- `data` 不能是空
- 行类型中必须包含 JSON 里声明的所有字段
- 行字段类型必须与导表 schema 一致

这意味着：

- 你不能随便写一个“差不多长得像”的类去加载
- 最稳妥的做法就是直接使用自动生成的 `XXXTable` 和 `XXXRecord`

### 9.2 缓存行为

`LoadData` 会缓存加载结果。

同一个类型、同一个表名重复加载时，会直接返回缓存对象，而不是重复读取资源。

对日常运行是有利的，但也要注意：

- 如果你在 Editor 下重新导出了 JSON，缓存会通过编辑器逻辑自动失效
- 如果你自己做了特殊的资源热更新流程，需要确认缓存是否符合预期

在 Editor 环境下也可以手动调用：

```csharp
LoadData.ClearCacheForEditor();
```

用于清空缓存。

## 十、全流程示例

下面给出一个从 Excel 到代码使用的完整流程。

### 10.1 准备 Excel

在 `Assets/Resources/Docs` 下创建：

- `Items.xlsx`

新增 `Item` Sheet，并填写：

| ID | 名称 | 价格 | 是否启用 |
| --- | --- | --- | --- |
| int | string | float | bool |
| id | name | price | enabled |
| 1 | Sword | 10.5 | true |
| 2 | Shield | 20 | false |

### 10.2 在 Unity 中导出

1. 打开 `Tools/Export Tables`
2. 选择 `Items.xlsx`
3. 点击“导出”

### 10.3 检查生成结果

应能看到：

- `Assets/Resources/GameData/ExportedTables/Items_Item.json`
- `Assets/Resources/GameData/ExportedTables/tables_manifest.json`
- `Assets/Scripts/GameData/GameDataTables.cs`

### 10.4 在业务代码中加载

```csharp
using UnityEngine;

public sealed class ItemDebugLoader : MonoBehaviour
{
    private void Start()
    {
        ItemTable itemTable = LoadData.LoadJSONData<ItemTable>("Item");
        if (itemTable == null)
        {
            Debug.LogError("ItemTable load failed.");
            return;
        }

        foreach (ItemRecord row in itemTable.data)
        {
            Debug.Log($"Item: {row.id} / {row.name} / {row.price} / {row.enabled}");
        }
    }
}
```

### 10.5 运行效果

进入 Play Mode 后，代码会：

1. 从 `tables_manifest.json` 查到 `Item` 对应的 JSON 路径
2. 从 `Resources/GameData/ExportedTables/...` 加载对应 `TextAsset`
3. 反序列化成 `ItemTable`
4. 从 `itemTable.data` 读取每一行 `ItemRecord`

## 十一、常见报错与排查

### 11.1 导出时报表头缺失

常见原因：

- 第 1 行、第 2 行、第 3 行中某一列缺了内容

当前工具要求每个有效列都必须同时具备：

- 描述
- 类型
- 字段名

只填了其中两项也会报错。

### 11.2 导出时报类型不支持

常见原因：

- 使用了工具未支持的类型

先改成当前支持的基础类型；如果确实需要复杂类型，需要后续扩展导表工具本身。

### 11.3 导出后没有生成某个 Sheet

常见原因：

- Sheet 少于 4 行
- 没有有效表头
- 没有任何数据行
- 整个 Sheet 被判定为应跳过

可优先查看工具窗口提示和 Unity Console 日志。

### 11.4 运行时加载返回 `null`

常见原因：

- 没有先执行导出
- `tables_manifest.json` 不存在
- 传入的表名不对
- 泛型类型不对
- 目标类型字段结构和导出 schema 不一致

优先检查：

1. `Assets/Resources/GameData/ExportedTables` 下是否已经生成 JSON
2. `GameDataTables.cs` 中是否存在对应的 `XXXTable`
3. 调用时传入的 `tableName` 是否等于 Sheet 名

### 11.5 修改 Excel 后运行结果没变化

优先检查：

1. 是否重新点击过“导出”
2. 是否导出了正确的文件或文件夹
3. Play 前是否已经完成 Unity 资源刷新

如果是编辑器调试中的特殊情况，也可以主动调用：

```csharp
LoadData.ClearCacheForEditor();
```

## 十二、协作注意事项

为了减少多人协作冲突，建议遵循以下约定：

1. 不要手改 `Assets/Scripts/GameData/GameDataTables.cs`
2. Excel 字段名一旦投入使用，不要随意改名
3. 新增表时，优先确认 Sheet 名是否符合业务命名约定
4. 提交 Excel 改动时，连同导出的 JSON 和 `GameDataTables.cs` 一起提交
5. 如果删除了某张表，也要执行一次导出，让旧 JSON 和 manifest 索引同步清理

尤其第 4 点很重要。当前运行时读的是导出的 JSON 和 manifest，而不是直接读 Excel。

## 十三、当前能力边界

当前导表工具的能力边界如下：

- 只支持 `.xlsx`
- 只支持基础标量类型
- 只支持以 Sheet 为单位导出
- 运行时加载依赖 `Resources`
- 自动生成的顶层类型是 `XXXTable`，行类型是 `XXXRecord`
- JSON 顶层是对象，不是纯数组

如果后续需要支持以下能力，需要额外开发：

- 枚举
- 数组或嵌套对象
- 多语言列规则
- Addressables 或外部热更资源
- 只生成行类型、不生成表包装类型

## 十四、相关文件

当前实现的主要代码位置：

- 导表工具：`Assets/Tools/ExportTablesTool/ExportTablesTool.cs`
- 运行时加载：`Assets/Scripts/Utils/LoadData.cs`
- 自动生成代码：`Assets/Scripts/GameData/GameDataTables.cs`

如果要修改导表规则、扩展字段类型、调整输出结构，通常都从以上文件入手。
