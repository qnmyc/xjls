# 游戏架构设计（Unity）

## 一、项目概览

本项目包含：

* 多章节（2~3章）
* 场景探索 + 交互
* 对话系统
* 背包系统
* UI 多层叠加

---

## 二、总体架构设计

采用：

Persistent 常驻系统 + Additive 场景加载（章节/菜单）

---

## 三、Scene 架构

### Scene 列表

```
Scenes/
|-- Persistent.unity
|-- MainMenu.unity
|-- Chapter1_Main.unity
|-- Chapter2_Main.unity
`-- Chapter3_Main.unity
```

---

### Scene 加载流程

#### 启动流程

```
Persistent（启动）
  -> Additive 加载 MainMenu
```

#### 开始游戏

```
卸载 MainMenu
加载 Chapter1（Additive）
SetActiveScene(Chapter1)
```

#### 切换章节

```
卸载当前章节
加载新章节
SetActiveScene(新章节)
```

---

## 四、核心设计原则

### 1. 公共系统常驻（Persistent）

以下内容只存在一份，放在 Persistent：

* UI 系统（对话、背包、弹窗、设置）
* 各种 Manager
* 音频系统
* 存档系统

---

### 2. 章节内容独立

每个章节 Scene 只负责：

* 地图
* NPC
* 交互物
* 触发器

---

### 3. UI 与逻辑解耦

章节不直接操作 UI，而是通过 Manager：

```csharp
DialogueManager.StartDialogue(data);
InventoryManager.AddItem(item);
UIManager.ShowPopup("获得物品");
```

## 九、数据层设计

核心原则：数据与 UI 分离

### 示例：背包系统

```
InventoryManager（数据层）
  └── 存储物品数据

InventoryCanvas（表现层）
  └── 显示物品
```

---

### 数据常驻内容

* 玩家背包
* 已解锁物品
* 剧情进度

## 十一、SceneFlowManager 设计

负责统一管理：

* 场景加载与卸载
* 黑幕过渡
* 设置 Active Scene
* 章节初始化

---

### 示例流程

```
StartGame()
  -> Fade Out
  -> Unload MainMenu
  -> Load Chapter1
  -> SetActiveScene
  -> Fade In
```

---

## 十二、推荐开发流程

1. 搭建 Persistent Scene
2. 实现 UIManager 和 SceneFlowManager
3. 实现 MainMenu
4. 完成 Chapter1 最小可玩闭环
5. 实现对话、背包
6. 扩展后续章节

---

## 十三、总结

Persistent 负责系统
章节 Scene 负责内容
UI 只保留一份
状态机统一控制行为
